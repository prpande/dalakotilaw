# Blog Feature Design — Dalakoti Law Website

**Date:** 2026-03-24
**Status:** Draft

## Overview

Add a fully functional blog to dalakotilaw.com for publishing legal articles and insights. Blog posts are authored as markdown files, loaded at runtime (no Angular rebuild needed), with bilingual EN/HI support and automatic manifest generation via GitHub Actions.

## Goals

- Easy authoring: drop a markdown file, commit, push — blog is live
- Bilingual support consistent with the existing translation system
- Home page integration showing latest articles
- No changes to the existing Angular build/deploy process

## Non-Goals

- Tag-based filtering or search (can be added later)
- Comments or social sharing
- CMS or admin UI
- RSS feed

## Architecture Rationale

Blog content lives in `docs/blog/` (outside the Angular `src/` tree) so it is served directly by GitHub Pages at runtime. This is the key decision that enables the "no rebuild" workflow — new posts are just static files committed to `docs/`, bypassing the Angular build entirely.

## Content Structure

### File Layout

```
docs/blog/
├── posts/
│   ├── .gitkeep
│   ├── understanding-property-rights.en.md
│   ├── understanding-property-rights.hi.md
│   ├── debt-recovery-know-your-rights.en.md
│   └── debt-recovery-know-your-rights.hi.md
├── images/
│   └── property-rights-hero.jpg
└── index.json          ← auto-generated by GitHub Action (seeded as empty [])
```

An initial empty `index.json` (`[]`) is committed so the Angular app never gets a 404 when no posts exist yet. A `.gitkeep` ensures the `posts/` directory exists in git.

### Markdown Frontmatter

Each `.md` file includes YAML frontmatter:

```markdown
---
title: Understanding Property Rights in India
date: 2026-03-24
summary: A guide to property ownership laws and common disputes.
tags: [property, civil]
image: property-rights-hero.jpg
slug: understanding-property-rights
---

Article body in markdown...

![Inline diagram](../images/some-diagram.jpg)
```

- `slug` pairs `.en.md` and `.hi.md` files together
- `image` references a file in `docs/blog/images/`
- `tags` use practice area names (informational only — no filtering in v1)
- Inline images use relative paths to `docs/blog/images/`

### Generated Manifest (`index.json`)

```json
[
  {
    "slug": "understanding-property-rights",
    "date": "2026-03-24",
    "image": "property-rights-hero.jpg",
    "tags": ["property", "civil"],
    "en": { "title": "Understanding Property Rights in India", "summary": "A guide to..." },
    "hi": { "title": "भारत में संपत्ति अधिकारों को समझना", "summary": "संपत्ति..." }
  }
]
```

Sorted by date, newest first. Contains both language variants per entry.

## GitHub Action

**File:** `.github/workflows/blog-manifest.yml`

**Trigger:** Push to `master` when files in `docs/blog/posts/**` change.

**Infinite loop guard:** The path filter is restricted to `docs/blog/posts/**` only — the auto-committed `index.json` lives at `docs/blog/index.json` (outside `posts/`), so it does not re-trigger. Additionally, the commit message includes `[skip ci]` as a safety net.

**Steps:**
1. Checkout repo
2. Setup Node 20
3. Run `scripts/generate-blog-manifest.js`
4. Auto-commit `docs/blog/index.json` if changed (with `[skip ci]` in message)

**Manifest generator script:** `scripts/generate-blog-manifest.js`
- Reads all `.md` files in `docs/blog/posts/`
- Parses frontmatter using `gray-matter` (dev dependency)
- Validates required fields (`slug`, `date`, `title`, `summary`, `image`) — skips files with missing fields and logs a warning
- Groups entries by slug, merging language variants — omits the `hi` key entirely when no `.hi.md` exists for a slug (aligns with the optional `hi?` in the TypeScript interface)
- Writes `docs/blog/index.json` sorted by date descending

## Angular Components

### TypeScript Interfaces

```typescript
interface BlogPost {
  slug: string;
  date: string;
  image: string;       // filename only, e.g. "property-rights-hero.jpg"
  tags: string[];
  en: { title: string; summary: string };
  hi?: { title: string; summary: string };  // optional — may not have Hindi version
}

interface BlogArticle {
  post: BlogPost;      // metadata from the manifest
  htmlContent: string; // sanitized HTML rendered from markdown
  lang: string;        // language that was actually loaded (may fall back to 'en')
}
```

### BlogService (`src/app/services/blog.service.ts`)

- `getPosts(): Observable<BlogPost[]>` — fetches `/blog/index.json` (absolute path — content lives in `docs/blog/`, outside the Angular build output, so relative paths could break depending on base href)
- `getPost(slug: string, lang: string): Observable<BlogArticle>` — fetches the markdown file for a slug + language, parses to HTML
- Falls back to `.en.md` if the requested language variant does not exist (404 response). If both `.en.md` and `.hi.md` are missing (e.g., manifest stale after file deletion), treats as "article not found" error.
- On manifest fetch error (network failure), returns empty array as fallback
- **Rendering pipeline:** `marked.parse(markdown)` → `DOMPurify.sanitize(html)` — always in this order
- **Image path resolution:** Thumbnail URLs constructed as `blog/images/{image}` from the manifest's `image` field. Inline markdown images use relative paths (`../images/...`) which resolve correctly from `/blog/{slug}`

### Blog Listing Page (updated `BlogComponent`)

- **Route:** `/blog`
- Fetches post list from BlogService
- Renders cards in reverse chronological order
- Each card: hero image thumbnail, title, date, summary, tags
- Title and summary use current language from the manifest (no markdown fetch needed)
- Language reactivity via `async` pipe in template (no manual unsubscribe needed)
- Shows `blog.no_posts` message when post list is empty
- Click navigates to `/blog/:slug`

### Blog Article Page (new `BlogPostComponent`)

- **Route:** `/blog/:slug`
- Fetches markdown for current slug + active language via BlogService
- Renders hero image full-width at top
- Renders article title, date, tags below hero
- Renders sanitized markdown HTML as article body
- "Back to Blog" link at top
- Re-fetches markdown when language changes (subscribes to `TranslationService.lang$`); unsubscribes on destroy to prevent memory leaks
- Shows fallback notice if Hindi version unavailable: "This article is not yet available in Hindi"
- **Loading state:** Shows a spinner while markdown is being fetched
- **Error state:** If slug does not exist at all, shows "Article not found" with a link back to blog listing

### Routing Updates (`app-routing.module.ts`)

Add routes:

```typescript
{ path: 'blog', component: BlogComponent, pathMatch: 'full' },
{ path: 'blog/:slug', component: BlogPostComponent },
```

`pathMatch: 'full'` on the blog route prevents it from matching `/blog/some-slug` via prefix matching.

## Home Page Integration

### Latest Articles Section

- Inserted between practice area cards and the CTA section
- Section heading: "Latest Articles" (via `home.blog.title` i18n key)
- Shows the **3 most recent** posts from `index.json`
- Cards are **visually smaller** than practice area cards:
  - Shorter image thumbnail height
  - Smaller font sizes
  - Tighter padding
- 3-column grid on desktop, stacks to 1-column on mobile (Bootstrap responsive grid)
- Each card: small hero thumbnail, date, title, summary (truncated)
- "View All Articles" button below cards → links to `/blog`
- Section hidden entirely if zero posts exist

### i18n Keys Added

```json
{
  "home.blog.title": "Latest Articles",

  "blog.no_posts": "No articles yet. Check back soon.",
  "blog.loading": "Loading article...",
  "blog.not_found": "Article not found"

```

Hindi equivalents added to `hi.json`.

## Bilingual Support

- **Listing page:** `index.json` contains both `en` and `hi` titles/summaries. Language switch swaps displayed fields without re-fetching.
- **Article page:** Language switch triggers re-fetch of the `.{lang}.md` file. Falls back to English with a notice if Hindi variant missing.
- **Slugs, image paths, tags** are language-agnostic (always English).
- Blog UI chrome (headings, buttons, labels) translated via existing `translate` pipe and i18n JSON keys.

## Publishing Workflow

To publish a new blog post:

1. Write `docs/blog/posts/{slug}.en.md` (and optionally `{slug}.hi.md`)
2. Add hero image to `docs/blog/images/`
3. Commit and push to `master`
4. GitHub Action auto-generates updated `index.json`
5. Post is live — no Angular rebuild needed
